How it works

From request to provisioned infrastructure in four steps.

1

Developer submits

Picks a template from the catalog and fills in parameters (instance type, region, tags…)

2

Approve or auto-approve

Requests within a team's cost threshold are approved automatically. Those that exceed it are queued for manual architect review.

3

GitOps commit

DeployCtrl renders Terraform files and pushes them to the configured GitHub repository.

4

Terraform runs

terraform apply runs in the background. Logs stream live to the dashboard.

Features

Everything you need to manage cloud infrastructure at team scale.

📋

Template catalog

Reusable infrastructure templates (EC2, S3, custom modules). Create, edit, and delete from the dashboard.

✅

Approval workflow

Submit → approve → provision. Per-team cost thresholds enable automatic approval for small requests.

🔀

GitOps integration

Rendered Terraform files are committed to a GitHub repository before every provisioning run.

⚡

Terraform automation

terraform apply and terraform destroy run in the background with live log streaming.

🗑️

One-click decommission

Runs terraform destroy, removes the GitHub folder, and clears state from MongoDB in one action.

🔐

Role-based access

admin, architect, developer, and user roles with per-permission granularity.

📜

Audit log

Immutable record of every platform action — who did what and when.

🗄️

HTTP Terraform backend

State stored in MongoDB. No S3 bucket or remote backend setup required.

Quick start

Up and running in under five minutes.

Docker Compose
Local install
# Clone and start
git clone https://github.com/jpmosquerac/deployctrl.git
cd deployctrl
docker compose -f local-dev/docker-compose.yml up --build
# Requires Python 3.11+ and MongoDB running locally
git clone https://github.com/jpmosquerac/deployctrl.git
cd deployctrl
bash local-dev/install.sh
source .venv/bin/activate
python manage.py runserver

API reference

All endpoints require Authorization: Bearer <token> except auth routes.

Method Path Description
POST/api/auth/login/Obtain access + refresh tokens
POST/api/auth/refresh/Refresh access token
GET / POST/api/templates/List or create templates
GET / PUT / DELETE/api/templates/<id>/Get, update, or delete a template
GET / POST/api/requests/List or submit requests
GET / PATCH/api/requests/<id>/Get or update (approve / reject) a request
POST/api/requests/<id>/retry/Re-trigger a failed Terraform run
POST/api/requests/<id>/decommission/Destroy and decommission a request
GET/api/requests/<id>/outputs/Get Terraform outputs for a request
GET/api/terraform/runs/List Terraform runs
GET/api/terraform/runs/<id>/logs/Get full log for a run
GET/api/audit/Query the audit log
GET / POST/api/teams/List or create teams
GET / PUT/api/settings/gitops/Get or update GitOps config
GET/api/users/List users
PATCH / DELETE/api/users/<id>/Update or deactivate a user
